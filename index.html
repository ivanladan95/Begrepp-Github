Begrepp och Github

Gitflow flow:

Flödet eller ”flowen” i git brukar kallas för gitflow. Med flödet så kan man säga hur vi med filerna vi skapat skickar, vad vi gör och våra kommandon. Strukturen som finns på Git och Githube finns av en bra anledning och är skapad av programmerare för programmerare. När man väl jobbar och lär sig denna struktur som finns i Git och Github så är den smidig och väldigt användningsbar för kodhantering och man inser snabbt att det är ett värdefullt verktyg. Git och Gitflow är inte det enda programmet som tar hand om ett kodhanteringssystem men det är ett av de vanligaste och är smidigt att jobba med. I Gitflowet man brukar prata om så finns det en rekommendation om hur man använder flödet i det, men detta flöde behöver dock inte följas det är bara ett standardflöde. Ett sätt som är smidigt när man jobbar flera med ett projekt är:

  1. Man startar i Github, sätter strukturen och delar upp uppgifterna
  2. Kopiera länken när du skapat ett repo. Eller om projektledaren lägger till dig i gruppen så skickas en länk till dig.
  3. På githube när du klickar på ”issues” kan du se vad projektledaren har tilldelat dig för uppgift ex. skapa en sida om Ivan. Nästa steg är att ”clona” ner det och skapa en egen bransch i din Vs code. Det är bra att namnge din bransch ett namn så alla i gruppen förstår vad det handlar om ex ivanladan-info.
  4. När du är klar så pushar du upp detta till repot och för att projektledaren ska kunna se och accpetera det du gjort så måste en ”Compare and pull request” på Githube göras. Nu lägger du till projektledaren i ”reviews” och du kan lämna en kommentar på det du gjort ex har gjort min info sida.
  5. Projektledaren kollar så att allt stämmer och att ingen ”merge-conflict” skett ex: om det skulle vara så att en linje med samma adress och mediatyp finns i både projekten. Om allt ser bra ut ger projektledaren dig ett godkännande.
  6. Nu kan du göra en ”merge” och lägga till detta i ert gemensamma projekt. Det är den som har bett om en ”pull request” som kommer göra en ”merge” när den fått ett godkännande av projektledaren.

Flexbox:

Idag är det ganska normalt att vi använder oss av massa olika skärmar typ som datorn, mobilen, Ipad mm. Det betyder att jag som webbutvecklare vill att mina sidor alltid ska fungera vart än du öppnar dem. Detta kan göras med hjälp av ”flexbox” som är en integrerad modul av css som löser problemen med att få en flexibel och responsiv layout så att sidan anpassar sig vart du än öppnar den. Så som namnet ”Flexbox” är det till för att flexibelt hantera ”boxars” eller lådors placering och storlek. Det är mycket lättare att få en layout som är bra med hjälp av denna teknik. Om du vill ha ”flexbox” funktionen och att din sida ska vara responsiv kan du skapa en div-class och namnger den ”container” exempelvis. Och nu kan du skapa fler div-classer där du har med box1, box2, och box3. För att aktivera funktionen måste du i sin css ha med din container och sedan skriva ”dispay: flex;” sedan är det bara att styla sina boxar i css med höjd, bredd, färg mm. Du kan också lägga boxarna hur du man vill och hur mycket plats dem ska ta på varje kolumn eller rad. Man kan säga med ”flexbox” kan man helt simpelt göra rutmönster och i rutorna lägga in innehållet man vill.

DOM:

Dom är ett sätt som förklarar att alla webbsidans delar är objekt. Man arbetar vanligtvis med olika språk så som html, javascript och html och de måste fungera tillsammans även om språken är olika och det är här DOM kommer in i bilden. Med Javascriptet kan du ändra HTML sidans design med hjälp av vissa funktioner och få grejer att dyka upp på vår html sida utan att man själv behöver uppdatera. Html och Dom är likadana och om några förändringar sker i html så ändras det också automatiskt i DOM.

Fin kod:

En strukturerad kod som är enkel att läsa och förstå. När man jobbar med kod så kanske det går en tid innan man sitter med samma kod igen det därför viktigt att ha en så strukturerad och lättläst kod som möjligt för att kunna förstå den igen och återuppta där man slutade. Man ska kanske inte förkorta namnen i variablerna man använder för det är smidigare för det är lätt hänt att du inte förstår vad MT är även om du menar matteTal. Om man i början skapar sig en god vana att strukturera koden medan du arbetar med den och lägger till kommentarer där man beskriver vad man gör för att underlätta för sig själv och andra om ni skulle jobba i samma projekt. Att hoppa in från kanterna när du skriver en funktion och lämna mer mellanrum kan vara en bra grej om man vill ha en mer lättläslig kod.

Ramverk:

Ramverk kan man säga är en tilläggstjänst som du kan lägga på ett programmeringsspråk som gör det mer enkelt och effektivt att jobba med programmeringsspråket. Inom programmering använder man verktyg som ex funktioner och kommandon och vid användning av ett ramverk kan utvecklarna välja mellan funktioner och kommandon som redan existerar och lägga in i sin egen kod för att bli mer effektivare och spara tid. Det är viktigt att hitta ett ramverk som du tycker passar dig och som kommer hjälpa dig.

Ett ramverk som skapades av Facebook-utvecklare som lanserades 2013 och idag är det ett ramverk som används av stora företag runtom världen. Det som gör React speciellt är att det är ett ramverk av Javascript. Man har lagt upp och strukturerat koden i komponenter som gör den mycket lätt att använda. Ramverket React blev snabbt populärt pga att den har en bra dokumentation, den vidareutvecklas aktivt, snabba bugg-fixar och den stora verktygslådan som medföljer. React har ett ramverk av Javascript som jag tagit upp innan och detta kan få vissa funktioner att hamna utanför som exempelvis datorhantering och då måste andra ramverk tas in för att samarbeta med React. Ramverket React är som sagt populärt och håller hög prestanda som används av många.

Ett annat ramverk som är ganska populärt är Vue som släpptes 2015 av Evan You. Ramverket vue som också är ett javascript ramverk och som skapades för att kunna skapa mer lättarbetade och kraftfulla gränssnitt. Den kommer med de vanligaste funktionerna och verktygen men inte i den stora grad som React kommer i, det är därför Vue är en av dem lättare ramverken att använda samtidigt som den erbjuder en lättarbetad och en effektiv utvecklingsmiljö. I Slutänden så ska man kolla på vilket ramverk passar dig och ditt projekt.
